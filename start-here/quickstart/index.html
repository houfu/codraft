Quickstart

This guide walks you through preparing the bundled Bonterms Mutual NDA using Claude Cowork. You will go from zero to a completed Word document in under five minutes.

Note

Codraft ships with the Bonterms Mutual NDA as an example template. It is a real, production-quality mutual NDA released under CC BY 4.0.

Get a Claude account and download the Desktop app

Section titled “Get a Claude account and download the Desktop app”

Note

Skip this step if you already have a Claude Pro or Max account and downloaded the Desktop App.

  1. Go to Claude and get an account. You will need at least Claude Pro (USD20/month) in order to use Cowork.

  2. Download the Claude Desktop App for Windows or Mac. Follow the instructions to install the app in your computer and connect it to your account.

Download the latest Codraft Folder

Section titled “Download the latest Codraft Folder”

There are several ways to use Codraft. For this quickstart, download the latest release:

⬇ Download codraft-latest.zip

Extract the contents to a place where you can find it (Downloads or Documents).

Note

Don’t download the plugin file. The plugin is used for Claude Code and other plugin marketplaces.

    Once Claude receives the task, it matches your request to the Bonterms_Mutual_NDA template, analyzes it for variables, and starts the interview. In the background, Claude installs the dependencies it needs and reads the templates. It looks for variables and creates a plan on how to ask and group questions in order to fill the template.

Note

The first time you run Codraft on a template, Claude is running code to analyse the template in detecting variables and generating questions. This file is saved in the template folder, speeding up future runs (at least until the next time you adjust the template.)

What just happened

Section titled “What just happened”

Behind the scenes, Codraft ran three steps:

  1. Discovery — Claude matched “mutual NDA” to the Bonterms_Mutual_NDA template directory
  2. Analysis — The Analyzer skill parsed the .docx template, extracted all 13 variables, and produced a manifest.yaml
  3. Rendering — After you confirmed the values, the Renderer skill used docxtpl to fill in the template and saved the output
